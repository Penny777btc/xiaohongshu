# xiaohongshu · 小红书选题 / 标题 / 封面 / 内容策略判断库

> 一个给 **做社媒视频、要投小红书（或同逻辑竖屏平台）** 用的 Claude 技能(skill)：把爆款方法论，翻译成"这条该不该做、标题怎么起、封面摆什么、发布带什么"的可执行判断 + 发布前体检清单。
> 来源：厦九九《5小时吃透小红书（商业版）》（人民邮电出版社，2024）。

---

## 这是什么

它不是一套"小红书运营教程"，而是一套**内容决策标准**。

做视频时你大概率遇到过这种情况：**"这条数据为什么不行，但说不清是哪一步出了问题。"**
这个 skill 就是把"凭感觉拍、凭手感起标题"，变成**一条条能复述、能检查的规则**——让你（和 AI）用同一套标准去判断：

- 这个选题**有没有话题性**？是不是目标人群真正关心的？
- 标题**勾不勾人**？是不是看完就知道全部、没有点进去的理由？
- 封面/缩略图**第一眼抓不抓得住主题价值**？
- 发布时**埋没埋对关键词、带没带对话题**，系统能不能打准标签？

核心一句话：

> **流量不是玄学：选题定上限 → 封面标题定点击 → 内容定完播与互动 → 进更大流量池。**

---

## 它在视频里帮你做什么

| 制作环节 | 这个 skill 给你的判断 |
|---|---|
| **写脚本前 · 选题** | 这条该不该做？用「对标 / 热点 / 人群痛点 / 跨界迁移」四法判选题有没有流量上限；没话题性就别开拍，封面标题再好也救不回来 |
| **写脚本后 · 标题** | 双标题包装（封面标题 + 正文标题，差异化又呼应）；13 种爆款句式选型；高唤醒情绪、直呼受众、留悬念、上数字——别让人看完标题就知道全部 |
| **封面 / 缩略图** | 竖版 3:4 最大化、图片有信息量、标题关键词醒目；三种爆款封面样式；字体颜色 ≤3 种 → 具体排版交给 [layout-design](https://github.com/Penny777btc/layout-design) |
| **发布 / 标签** | 封面标题正文埋 1–3 个标志词让系统打对标签；带 1 类强相关 + 高浏览量话题；选活跃时段；结尾留钩子引导互动（评论权重高） |

外加一份**发布前体检清单**：把"看着没流量"定位成可改的项（选题没话题性 / 双标题偷懒 / 标题说尽 / 封面无信息量 / 开头铺垫太长 / 没埋关键词 / 没引导互动 / 踩合规线）。

## 核心法则 · 使用前 / 使用后

这个 skill 的价值，就是把"凭感觉做"换成"按链路判断"。以下几条是最常用、最容易立竿见影的：

### 01 · 选题定上限

- **使用前**：选题靠拍脑袋，挑自己想拍的，结果是个没人关心的主题。封面标题做到 80 分，数据照样惨。
- **使用后**：先跑对标 / 热点 / 痛点三法，唯一判据是"目标人群关不关心、有没有话题性"。没话题性当场换题，不浪费后面的工。
- **为什么**：一个 80 分选题配 60 分文案能爆；一个没人关心的选题，文案 80 分也死。选题是流量的天花板。

### 02 · 双标题，别写一样

- **使用前**：封面标题和正文标题写成一句话，白白浪费一个广告位。
- **使用后**：封面标题主打亮点 A、正文标题主打亮点 B，差异化又呼应，关键词层层叠加价值感。
- **为什么**：发布时本来就有两个标题位，写成两个不同卖点 = 两次抓人机会。

### 03 · 标题别说尽

- **使用前**：标题把内容全交代了，看完没有点进去的理由。
- **使用后**：留悬念 / 留钩子 / 埋反常识，给一个"必须点进来"的缺口。
- **为什么**：最忌让人看完标题就知道全部——那是失败标题，点击率的第一道闸就关上了。

### 04 · 封面要有信息量

- **使用前**：封面模糊 / 居中呆板 / 没重点，被系统判低质，第一眼抓不住人。
- **使用后**：竖版 3:4 最大画幅、图片清晰有场景感、封面标题关键词醒目、字体颜色 ≤3 种。
- **为什么**：封面 + 标题决定点击率；模糊或无信息的封面会被判低质而不推荐。版面怎么排得高级，交给 [layout-design](https://github.com/Penny777btc/layout-design)。

### 05 · 开头别铺垫

- **使用前**：开头一大段铺垫和寒暄，5 秒内没给重点，用户提前划走，伤完播也伤后续推荐。
- **使用后**：亮点前置、价值先行、开门见山——一两句直击重点。
- **为什么**：完播率是第二道闸；开头 5 秒决定去留，离开趋势会被系统读成"内容不行"。

### 06 · 埋对关键词、带对话题

- **使用前**：不埋关键词、不带话题或乱带弱相关话题，系统打不准标签，搜索流量为零。
- **使用后**：封面 / 标题 / 正文 / 话题埋 1–3 个高频但差异化的标志词，带 1 类强相关 + 高浏览量话题。
- **为什么**：标签打准才进对的流量池；关键词布局是长尾搜索流量的根（半年后仍被搜到）。

### 07 · 结尾引导互动

- **使用前**：内容讲完就结束，不给关注理由、不抛问题，互动率低、涨粉少。
- **使用后**：结尾升华 + 金句 + 固定口号 + 抛问题主动引导评论。
- **为什么**：CES 里评论 / 转发权重 ×4、关注 ×8 > 点赞收藏 ×1；互动率 10–30% 大概率爆，<3% 难起。

### 08 · 别踩合规线

- **使用前**：简介放微信号导流 / 晒产品图当内容 / 踩高风险类目 / 收赞助不声明——轻则限流重则封号。
- **使用后**：站外导流改走平台合规工具、"晒产品"改成"内容分享"、收赞助主动声明。
- **为什么**：这是发布前体检的最后一关，命中就改写或撤下，否则前面做得再好也进不了流量池。

## 关注度 vs 可信度（边界里的一条线）

封面标题越夸张高唤醒 = 点击率越高但越像标题党；越克制真实 = 可信但易被划走。
小红书生态吃"真实分享"，**标题封面可适度博点击，但内容必须接得住、别名不副实**——名不副实会拉低完播和互动，反而被停推。按视频调性定档。

---

## 它不做什么（边界）

- **不替代脚本文案本身**：怎么把文案写出来、遣词造句，归脚本环节。它只管"该做什么选题、用哪种结构、起哪种标题、封面放什么信息"。
- **不替代封面排版执行**：字级主次、配色克制、版面怎么排得高级，交给配套的 [layout-design](https://github.com/Penny777btc/layout-design)。本 skill 只管"封面该放什么、用哪种样式"。
- **不是小红书运营教程**：不讲涨粉投流、商业变现、矩阵号那一套。聚焦"内容怎么被点开、看完、推荐出去"这条链路。
- **不是硬规则**：判断服务于内容与目的，视频调性优先（见上面"关注度 vs 可信度"）。
