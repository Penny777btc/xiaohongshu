# 小红书内容策略（判断库）

来源：厦九九《5小时吃透小红书（商业版）》。本 skill 把书里"流量获取—内容生产—内容发布"的爆款方法论，
转译为**本频道做社媒视频时的选题/标题/封面/发布判断**——不是教你做小红书运营，而是当一条视频要投小红书（或同逻辑的竖屏平台）时，帮你在"写脚本之前定方向、写脚本之后做包装"两个节点上做对决策。
深度原理与公式见 [reference.md](reference.md)；本文件只放**何时用、怎么判断、怎么验收**。

## 核心前提（先记住这一条）

**选题 > 标题封面 > 内容本身**，对流量的决定顺序如此。
一个 80 分选题配 60 分文案能爆；一个没人关心的选题，文案 80 分也死。
而决定"点不点进来"的是**封面 + 标题**（点击率），决定"看不看完、推不推荐"的是**内容**（完播率 + 互动）。

> 流量不是玄学：点击率（封面标题）→ 完播率（开头+节奏+价值）→ 互动率（共情+争议+引导）→ 进更大流量池。任一环掉链子，推荐就停。

跟 [[write-script]]、[[layout-design]] 是一组：本 skill 管"做什么选题、起什么标题、封面摆什么信息"，write-script 管脚本怎么写，layout-design 管封面/花字版面怎么排得高级。

## 何时调用 / 接入点

| 节点 | 本 skill 提供的判断 |
|---|---|
| **写脚本前·选题** | 这条该不该做？用「对标/热点/人群痛点/跨界迁移」四法判选题有没有流量；选题要有"话题性"，否则封面标题再好也白搭 |
| **写脚本后·标题** | 双标题包装（封面标题 + 正文标题，差异化呼应）；13 种爆款标题句式选型；高唤醒情绪、直呼受众、留悬念、上数字 |
| **封面 / 缩略图** | 竖版 3:4 最大化、图片有信息量、标题关键词醒目；三种爆款封面样式；字体颜色≤3 种 → 具体排版交 [[layout-design]] |
| **发布 / 标签** | 标题正文埋关键词让系统打对标签；带 1 类强相关 + 高浏览量话题；选活跃时段；结尾留钩子引导互动 |

## 分模块速查

**① 选题逻辑（决定流量上限）**
- 四种找选题法：**对标追踪**（盯 5–10 个比你早几个月、量级相近的同行近 3 天爆款，可驾驭就跟，选题可像内容必须差异化）、**热点借势**（时事型快但避争议/涉政；周期型靠营销日历提前蹭）、**人群痛点**（画用户像→推痛点→搜高赞问题，问题即选题）、**跨界迁移**（把别领域的爆款句式/模式嫁接到本频道）。
- 维护**选题库**：在线文档随手记、用过打钩、没灵感来挑。选题灵感也来自粉丝留言、爆款评论区高赞评论、公众号爆文标题。
- 判据：**这个选题有没有话题性、是不是目标人群关心的**。没有就别开拍。

**② 爆款标题公式（决定点击率）**
- **双标题原则**：封面标题（短、亮点 A）+ 正文标题（≤20 字、亮点 B），二者**差异化又呼应**，等于两个广告位，别写一样的。
- 13 种句式按需选：情绪代入（真心建议…/后悔没早点…）、直呼受众（植入身份职业年龄→对号入座，还能让推荐更精准）、提问/设问、悬念（说一半留一半）、数字（大人物用大数、要用户行动用小数，一题≤3 个数字）、利益、对比、颠覆认知、解决痛点、借力（名人/名牌/热点背书）、共鸣（多用"你""我"）、命令、概括。
- 铁律：**别让人看完标题就知道全部**——没有点进去的理由就是失败标题。

**③ 封面要点（决定点击率）**
- **竖版 3:4**（双列信息流最大画幅）；图片清晰美观有信息量、有场景感/氛围感（模糊会被判低质不推荐）；**封面标题/关键词醒目**让人一眼抓住主题价值。
- 封面排版万能式 = 竖版/上下二分 + 场景化/纯色背景 + 人物/物品主体 + 标题/关键词。三种爆款样式：人物主体+一句话、二分拼图+居中标题、人物照片+上下标题。**字体颜色≤3 种**。
- **专属封面坚持用**→主页整洁统一（频道一致性）。具体版面细节交 [[layout-design]]。

**④ 笔记结构（决定完播/完读）**
- 短视频文案万能式：**抓人开头 + 高密度有价值正文 + 升华结尾**。开头一两句直击重点别铺垫（5 秒内决定去留）；正文围绕 2–5 个要点、信息密度高、口语化、专名即时解释；结尾升华+金句+引导关注（如固定口号）。
- 五大可套结构：结论先行（讲观点）、问题解决（分享干货/带货）、清单体（好记）、故事体（最打动人）、短视频框架（通用口播）。详见 reference。
- 正文描述别复制全文案：提纲挈领 + 留悬念钩子，否则没人看视频，伤完播。

**⑤ 流量/推荐机制（决定能不能进更大池）**
- 链路：审核通过 → 打标签小范围测（如 1000 曝光）→ 数据达标进下一级池（5000/1w…），不达标停推。
- 关键指标：**点击率**（封面标题）、**完播率/完读率+停留时长**（内容质量，目标高于同类 60–90%）、**互动率**（点赞+收藏+评论 ÷ 观看，10–30% 大概率爆，<3% 难起）。CES 里**评论/转发权重×4、关注×8** > 点赞收藏×1 → 刻意引导评论、给关注理由。
- 搜索流量是长尾：**关键词布局**（封面/标题/正文/话题埋 1–3 个高频但差异化的标志词）+ 用下拉扩展词/社区热搜词选词。

**⑥ 合规（决定会不会被限流/封号）**
- 高压线：站外大量导流（微信号/电话放简介头像笔记）、纯营销广告/晒产品图当内容、违规词极限词、搬运抄袭洗稿、未声明的利益关系。
- 高风险限流类目：医美/宣称功效/减肥药、荐股基金期货、招商加盟/赚钱 App——能发但几乎没流量。
- 思维转换：把"晒产品"改成"内容分享"；收到赞助须声明；尽量站内成交。

## 发布前体检（逐条过，命中即改）

- [ ] **选题没话题性** — 不是目标人群关心的 → 换选题，别靠封面标题硬救。
- [ ] **双标题偷懒** — 封面/正文标题写一样 → 拆成差异化呼应的两个亮点。
- [ ] **标题说尽了** — 看完标题就知道全部 → 留悬念/留钩子/埋反常识。
- [ ] **封面无信息量/模糊/居中呆板** — → 上 3:4、清晰有场景、标题醒目；版面问题转 [[layout-design]]。
- [ ] **开头铺垫太长** — 5 秒内没给重点/价值 → 亮点前置、开门见山。
- [ ] **正文复制全文案** — 没悬念 → 提纲挈领 + 留钩子。
- [ ] **没埋关键词/没带话题** — 系统打不准标签 → 封面标题正文话题埋 1–3 个标志词 + 1 类强相关高浏览话题。
- [ ] **没引导互动/没给关注理由** — → 结尾抛问题+固定口号；评论权重高，主动引导评论。
- [ ] **踩合规线** — 站外导流/广告感/高风险类目/未声明赞助 → 改写或撤下。

## 关注度 vs 可信度（同 [[shot-language]]/[[layout-design]]）

封面标题越夸张高唤醒 = 点击率越高但越像标题党；越克制真实 = 可信但易被划走。
小红书生态吃"真实分享"，**标题封面可适度博点击，但内容必须接得住、别名不副实**——名不副实会拉低完播和互动，反而被停推。按 brief 调性定档。

## 进化闭环

本 skill 是**原理依据**；当某条选题/标题/封面被数据验证（点击率、完播、互动）或被用户确认好/纠正，按管线惯例**提炼成一条规则补进 `pipeline/taste.md`**（选题/标题/封面节），让判断沉淀为本频道可复用资产。本文件承载"为什么这样选/这样起"，taste.md 承载"本频道实际怎么做、哪些选题哪些句式被验证有效"。
